# Lead v2 Hack Detection

## When: Phase 5 Review, Round 1, in parallel with Codex/critic
## When NOT: standalone code review outside /leadv2 flow; security auditing (use security-auditor)

## Input

Read the diff file at the path provided in the mission (e.g. `/tmp/leadv2-review-<task-id>.diff`).
Parse only `+` lines (additions) from the diff — do not flag deletions or context lines.

## Signal catalog

Detect patterns: `todo-no-ticket`, `magic-number`, `broad-except`, `disabled-test`, `special-case`, `commented-code`, `hardcoded`.

For full pattern definitions and detection heuristics, see [SIGNALS.md](./SIGNALS.md).

## Severity rules

- `block`: disabled test without justification. Hard-wired, not opinion. Only this case.
- `warn`: all other patterns — quality signal, not a blocker.
- `info`: reserved for future use.

## Output format

Write findings to `docs/handoff/<task-id>/hack-findings.yaml`:

```yaml
# hack-detection findings — generated by leadv2-hack-detection
task_id: <task-id>
diff_file: /tmp/leadv2-review-<task-id>.diff
scanned_at: <ISO timestamp>
findings:
  - type: todo-no-ticket
    file: platform/agent/foo.py
    line: 42
    snippet: "# TODO: remove this after migration"
    severity: warn
    suggestion: "Link a ticket or remove — untracked TODOs accumulate as silent tech debt"
  - type: disabled-test
    file: tests/test_bar.py
    line: 15
    snippet: "@pytest.mark.skip"
    severity: block
    suggestion: "Add reason= argument or restore the test — skipping without reason hides regressions"
summary:
  block_count: 1
  warn_count: 2
  info_count: 0
  has_block: true
```

Also write a one-line summary to `docs/handoff/<task-id>/hack-detection.summary.md`:
```
Hack-detection: <N> block, <N> warn findings. [block: <type list>] Full: hack-findings.yaml
```

## Rules

- Only scan `+` lines — never flag existing code that wasn't changed in this diff.
- `block` is hard-wired to `disabled-test` without reason only. Do not escalate warns to blocks based on count.
- Write findings atomically: write to a temp path, then mv to the final path.
- Empty findings list is valid — emit `findings: []` and `summary.block_count: 0`.
- Do NOT read full files unless the diff line is ambiguous about surrounding context (e.g., to confirm broad-except has no re-raise on next line).

## Implementation

Use Python stdlib only (no third-party imports). For regex patterns, Python code examples, parsing instructions, and anti-patterns, see [IMPLEMENTATION.md](./IMPLEMENTATION.md).

## Integration with lead-reflect

After reading hack findings, lead-reflect uses `hack_findings` to compute `fix_quality`. See [IMPLEMENTATION.md §Integration](./IMPLEMENTATION.md#integration-with-lead-reflect) for the quality-mapping rules.
